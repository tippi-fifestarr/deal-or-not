### 4. Privacy angle is missing — key hackathon track

The CRE FTW section should lead with privacy/confidentiality:

- **Gemini API key stays inside the enclave.** Confidential HTTP means the DON nodes never see the key — it's injected at runtime inside the secure enclave. The Banker's personality comes from Gemini 2.5 Flash, but the API key is never exposed to the network.
- **Case values are "quantumly undecided."** The formula is `hash(vrfSeed, caseIndex, usedBitmap, creEntropy)` where `creEntropy` is fetched at runtime via Confidential HTTP from an external randomness source. Nobody — not the player, not the contract, not even the CRE node operator — knows the case value until the enclave computes it. The VRF seed is committed on-chain at game creation, but the case values don't exist until they're opened.
- **The DON sees nothing.** All computation happens inside the CRE confidential enclave. WriteReports deliver results to the chain without exposing intermediate values.

### 5. Two writeReports vs separate workflows

In the **prototype** (`prototype/workflows/`): the banker workflow does 2 writeReports in a single CRE simulation — one for the game offer (`setBankerOfferWithMessage`), one for the BestOfBanker gallery (`saveQuote`). This causes nonce collisions in simulate mode.

In **PR #16 convergence** (`packages/convergence/workflows/`): the quote save is a **separate workflow** (`save-quote/main.ts`), which fixes the nonce collision. PR #16 has 4 workflows total:
- `confidential-reveal/main.ts`
- `banker-ai/main.ts` (offer only)
- `save-quote/main.ts` (gallery save — split out)
- `sponsor-jackpot/main.ts`

Plus PR #13 added the `agent-gameplay-orchestrator` as a 5th CRE workflow.

**After PR #16 merges, update the card text to reflect the split workflow architecture.**
